Unit 10: Capstone Outline

As well as the final deadline for the Unit 10 summative, we have been asked to consider starting to formalise our thoughts regarding the capstone project by considering the following areas:

Research area and working title.

As the course has progressed, I have developed an interest in encryption and the unique problems of its implementation in resource constrained Internet of Things (IoT) environments. My working title is:

Investigating the efficacy of lightweight encryption in resources constrained IoT environments.

Proposed Research Problem and Research Question.

R1 – What metric of measuring the efficiency of the algorithms best reflects this use case?

R2 – Which algorithm is most efficient for the proposed use case?

The results of my research could help in the selection of an encryption algorithm most appropriate for a particular use case.

Proposed Aims and Objectives.

To develop a system to measure efficiency. Current thoughts are this could include components of encryption time (CPU cycles? Code execution time?), energy usage, memory usage and encryption strength.
Bharathi, et al., (2017) suggest the following for measuring the performance of an encryption algorithm:
  • encryption time / decryption time
  • throughput of encryption / throughput of decryption
  • CPU process time
  • CPU clock cycles
  • power consumption
  • memory use
The weighting of each measure will need to be carefully considered for resource constrained IoT environments and a particular use case. For example, for a remote sensor relying on battery power, power consumption will have a higher weighting than a case where the sensor has mains power.

Proposed Methodology.

  • Create testing bed.
  • Decide on measure of efficiency.
  • Select algorithms [based on Bouting, (2023)?]
  • Test algorithms to collect data.
  • Process and analyse the data to answer R1 & R2.

Artefact(s) that can be created.

  1. A physical testing environment composed of sensors, broker and controller(s).
  2. The Python code required to implement each encryption algorithm over the MQTT protocol between sensor / controller / broker / subscriber.

References.

Bharathi, B., Manivasagam, G. and Kumar, M.A., (2017) Metrics for performance evaluation of encryption algorithms. Int J Adv Res Sci Eng, 6(3), pp.62-72.

Bouting, C. (2023) NIST Selects ‘Lightweight Cryptography’ Algorithms to Protect Small Devices. Available from: https://www.nist.gov/news-events/news/2023/02/nist-selects-lightweight-cryptography-algorithms-protect-small-devices [Accessed 25th June 2024].
